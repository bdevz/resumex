# Anti-Slop Prompt Hardening — Design Spec

## Problem

Resume bullets generated by the system often exhibit telltale signs of AI-generated writing: inflated vocabulary ("spearheaded", "leveraged", "transformative"), formulaic structure (every bullet has a metric, adjective triplets), and an over-polished tone that experienced hiring managers and recruiters immediately recognize and distrust.

The current `scoreBullet()` function rewards presence of metrics, tech names, and strong verbs — but never penalizes AI tells. Some anti-slop guidance exists in the prompt builders, but it's scattered across four functions, hardcoded, and inconsistent.

## Goal

Build a configurable, prompt-based anti-slop system that:
1. Prevents AI-generated writing patterns from appearing in resume output
2. Lives in a single config block so rules can be added, removed, or adjusted without touching prompt logic
3. Applies consistently across all four prompt modes (standard, XL, optimize, optimize-XL)
4. Models the mental filter of a hiring manager who is tired of AI-generated resumes

The system is opinionated: it prioritizes sounding authentically human over sounding impressive. Even if it means a less "polished" resume, authenticity wins.

## Approach

**Prompts-only defense** — anti-slop rules are injected into the system prompts sent to the AI. No changes to post-generation scoring. This keeps the feedback loop simple: iterate on config, regenerate, evaluate.

**Config-driven** — all rules live in a single `ANTI_SLOP` object in `config.js`. A single function `buildAntiSlopPromptSection(mode)` in `prompts.js` reads the config and generates the prompt text. The four prompt builders each call this function.

**Severity levels** — each rule has a `severity` field:
- `"hard"` → "NEVER use this" in the prompt
- `"soft"` → "Avoid when possible" in the prompt

This allows softening rules without removing them.

## Design

### 1. `ANTI_SLOP` config block in `config.js`

**Banned words** — single words that signal AI writing:
- Hard: "delve", "pivotal", "underscore", "landscape", "foster", "testament", "leverage", "streamline", "spearhead", "harness", "elevate", "bolster", "utilize", "tapestry", "intricate", "groundbreaking", "transformative", "innovative", "revolutionize", "synergy", "paradigm"
- Soft: "robust", "seamless", "comprehensive", "cutting-edge", "world-class", "best-in-class", "holistic"

**Banned phrases** — multi-word patterns:
- Hard: "not just X, but Y", "plays a vital role", "stands as a testament", "it's important to note", "from X to Y" (rhetorical, not metric), "driving innovation", "ensuring seamless", "state-of-the-art", "in today's fast-paced", "at the forefront of"
- Soft: "end-to-end", "above and beyond", "key stakeholders"

**Structural anti-patterns** — each with name, description, bad example, good example, and severity:
- `adjective_triplet` (hard): Three adjectives in a row ("scalable, resilient, and performant")
- `trailing_ing_clause` (hard): Dangling -ing clause that adds no specifics ("...ensuring compliance across all departments")
- `em_dash_overuse` (soft): More than one em dash in a single bullet
- `every_bullet_has_metric` (hard): Every single bullet in a role has a percentage, dollar amount, or numeric comparison
- `metric_stacking` (hard): Multiple unrelated metrics crammed into one bullet

**Good vs bad examples** — three pairs with explanations injected into the prompt:
1. "Spearheaded a transformative cloud migration..." vs "Moved 14 services from EC2 to EKS over 4 months..."
2. "Drove innovation by implementing a robust microservices architecture..." vs "Broke a Rails monolith into 6 services so deploys stopped taking down the whole app every Thursday"
3. "Leveraged cutting-edge machine learning algorithms..." vs "Built a churn prediction model using XGBoost that flagged at-risk accounts 2 weeks before they cancelled..."

**Tone guidance:**
> Write like a tired engineer updating their resume at 11pm, not like a marketing copywriter. Be specific and plain. If something was hard, say it was hard. If the scope was small, don't inflate it. A human resume has personality — it mentions the annoying migration, the team that was skeptical, the workaround that became permanent. Never make every bullet sound triumphant. Real work is messy.

**Metric ratio constraint:**
- Standard mode: at most 4 of 6 bullets per role should have hard metrics
- XL mode: at most 7 of 15 bullets per role should have hard metrics

### 2. `buildAntiSlopPromptSection(mode)` in `prompts.js`

A single function that:
1. Reads `config.ANTI_SLOP`
2. Splits banned words/phrases by severity into NEVER vs Avoid lists
3. Selects metric ratio based on mode parameter
4. Assembles and returns a prompt string covering: tone, banned vocabulary, banned phrases, structural anti-patterns with examples, metric ratio, good/bad example pairs, and sentence structure variety guidance

Takes a `mode` parameter: `"standard"`, `"xl"`, `"optimize"`, `"optimize-xl"`.

### 3. Prompt builder changes

Each of the four prompt builders removes its scattered anti-slop guidance and replaces it with a single call to `buildAntiSlopPromptSection(mode)`.

**Removed from `buildSystemPrompt()`:**
- "Avoid excessive adjectives/adverbs" line
- Entire "WRITING STYLE — SOUND HUMAN, NOT AI-GENERATED" section
- "A resume that reads like a human wrote it stands out" line

**Removed from `buildSystemPromptXL()`:**
- Adverb avoidance line
- Entire "WRITING STYLE" section

**Removed from `buildOptimizeSystemPrompt()`:**
- Adverb avoidance line
- Entire "WRITING STYLE" section

**Removed from `buildOptimizeSystemPromptXL()`:**
- Adverb avoidance line
- Entire "WRITING STYLE" section

**Kept untouched in all prompt builders:**
- CRITICAL RULES (JSON format, schema, verb lists, timeline constraints)
- WORK HISTORY GUIDELINES
- BULLET ORDERING / SKIMMABILITY section
- BUSINESS VALUE MIX section
- COMPANY & DOMAIN CONTEXT section
- TECHNOLOGY TIMELINE section
- "Use digits instead of spelling out numbers" (formatting, not anti-slop)

## Files changed

| File | Change |
|------|--------|
| `lambda/lib/config.js` | Add `ANTI_SLOP` block, add to `module.exports` |
| `lambda/lib/prompts.js` | Add `buildAntiSlopPromptSection(mode)`, remove scattered anti-slop from 4 prompt builders, insert function call in each |

No other files change. No changes to `scoreResume()`, `scoreBullet()`, `validateTimeline()`, or `lambda/index.js`.

## Iteration workflow

1. Read a generated resume
2. Spot an AI tell
3. Add the word/phrase/pattern to `ANTI_SLOP` in `config.js` with appropriate severity
4. Regenerate — the rule now applies across all four prompt modes
